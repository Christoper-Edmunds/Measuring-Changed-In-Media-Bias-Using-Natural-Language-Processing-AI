About this project

Overview

The goal of this project is to track changes in the sentiment of named key figures over time in the articles published by the BBC

What do you mean by a key figure?

By key figures we are referring to a specific subset of named entities including, People, Places, Organisations and "Groups" (E.g: Christians or Communists)

How does this website work?

The Website functions by following a few key steps

  • The newest articles from the bbc are fetched via a webcrawler
  • The text is broken up into sentences which are cleaned up to remove junk
  • Each sentence has its sentiment appraised using the Textblob sentiment analysis model
  • Each sentence has its named entities processed using the Spacy NER model
  • The relevant named entity is selected from the possible choices based on type of entity and position in the sentence
  • The average sentiment for each key figure is averaged out across the article
  • Key figures with overlapping names (E.g "Coal" and "Jacob Coal" are combined together
  • All key figures sentiment averages are weighted up or down based on the overall sentiment of the article
  • The finalised sentiments, key figures and date of article are entered into the database
  • Finally, this website searches the database for matching named key figures to the one you select, and displays it on the graph

What technologies does this website make use of?

The core technologies are, Python, Flask, Spacy, Textblob, Numpy, SQLite and BeautifulSoup

What are the limitations of this website?

The limitations of this website are quite significant, sentiment analysis is never 100% accurate and written language often follows complex and changing rules making objective assesment difficult, however as a tool to track generalised trends positive or negative, as well as provide insights about the general writing styles of media this tool can be relatively useful.
